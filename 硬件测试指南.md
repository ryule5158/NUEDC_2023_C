# 集成工程硬件测试指南

本文对应 `集成工程` 中的通用 FPGA、STM32H743 和 TI MSPM0G3507 工程。测试时一次只允许一块 MCU 驱动 BX71 SPI，不能把 STM32 与 TI 板同时接在同一组 P2-31～P2-34 上。

## 1. 能力边界

- AD9708 以 125 MSPS 在 FPGA 内自主产生恒定码、标准波形、任意波和扫频。
- AD9280 以固定 32 MSPS 在 FPGA 内采集，缓存 1～4096 点，支持 1～65535 倍抽取及四种触发。
- MCU 只配置 FPGA 并读取 BRAM，不实时接收连续 32 MSPS 数据。
- AD9280 没有触发前采样；阈值触发所得数据从触发后开始。
- ProMax 实时档直接处理 AD9280 的连续原始流，提供 8 路 DDC/CIC64 功率和 4 组 32 抽头匹配峰值；功率约每 2.048 ms 更新一次。
- 当前 ProMax 不实现 UART、SPI 线级 CRC 或 FPGA FFT。公共 FFT 接口保留用于能力探测，调用会明确返回 `FPGA_PROMAX_E_UNSUPPORTED`。
- TI 片上 ADC 当前为单通道、512 点、最高 20 kSPS 的软件定时单次采集；没有移植 STM32 的 DMA 连续模式，传入 `ADC_MODE_CONTINUOUS` 会返回错误。
- TI 的 AD5687/DAC8831 软件波形上限为 1000 点/秒；16 点表的频率上限约 62 Hz。高速 AD9708 不受此限制。

## 2. 公共硬件准备

1. 断电操作插接和跳线。
2. 高速 AD/DA 模块在 U3 位置与 BX71 P3 尾端对齐直插，不翻面、不错位。
3. 模块使用稳定 +5 V 供电；BX71 与 MCU 数字 IO 都是 3.3 V，禁止向逻辑引脚输入 5 V。
4. S1 是 AD9708 DAC 输出，S2 是 AD9280 ADC 输入。
5. BX71、MCU、模块、信号源和示波器必须共地。
6. 初次测 S2 使用正弦波 `100 kHz、1 Vpp、0 V 偏置`。信号源设为高阻负载显示模式；不确定时先用示波器确认实际幅度。
7. 不要为了测试 OTR 主动超过模块标称约 `-5～+5 V` 的输入范围。

FPGA 掉电启动应烧录：

`FPGA_BX71_AD9708_AD9280/output/BOOT_PS7.bin`

BX71 保持 QSPI 启动档。上电后 FPGA 应自动配置完成；只下载 `.bit` 不能掉电保存。

## 3. MCU 与 BX71 接线

### 3.1 STM32H743

| STM32H743 | BX71 | 作用 |
|---|---:|---|
| PI1 / SPI2_SCK | P2-31 | SPI 时钟 |
| PI3 / SPI2_MOSI | P2-32 | MCU 写 FPGA |
| PI2 / SPI2_MISO | P2-33 | FPGA 返回数据 |
| PE8 / FPGA_CS | P2-34 | 低有效软件片选 |
| PH9 | P2-35 | 可选 IRQ，基础测试可不接 |
| GND | P2-30 或 P2-12 | 共地 |

### 3.2 TI MSP-LITO-G3507

| TI 信号 | MSP-LITO位置 | BX71 | 作用 |
|---|---|---:|---|
| PA12 / SPI0_SCLK | J1-30 | P2-31 | SPI 时钟 |
| PA14 / SPI0_PICO | J1-32 | P2-32 | TI 写 FPGA |
| PA13 / SPI0_POCI | J1-31 | P2-33 | FPGA 返回数据 |
| PA15 / GPIO | J2-32 | P2-34 | 低有效软件片选 |
| GND | 任一 GND | P2-30 或 P2-12 | 共地 |

TI 板与 BX71 各自供电，只连四根逻辑线和 GND，不连接两板的 3V3/5V 电源线。以调试接口在上、按键在下的元件面观察：J1最下排左/右分别为PA13/PA14，J1倒数第二排右侧为PA12，J2最下排右侧为PA15；板卡接口位置可对照 [TI 官方 MSP-LITO-G3507 用户指南](https://www.ti.com/lit/ug/slvuda3/slvuda3.pdf)。

## 4. STM32 高速 ADC 测试

### 4.1 编译和下载

1. 打开 `STM32H743/MDK-ARM/STM32H743.uvprojx`。
2. 执行 Rebuild，必须是 `0 Error(s), 0 Warning(s)`。
3. 用当前工程原有下载器设置把程序烧入 STM32。
4. 确认第 3.1 节接线，信号源接 S2。

### 4.2 加入一次性测试代码

在 `Core/Src/main.c` 的 `USER CODE BEGIN Includes` 中临时加入：

```c
#include "ad9280_app.h"
```

在 `USER CODE BEGIN PV` 中临时加入：

```c
volatile AD9280_StatusTypeDef g_adc_init_status;    /* ADC初始化状态。 */
volatile AD9280_StatusTypeDef g_adc_capture_status; /* ADC采集状态。 */
const AD9280_AppDataTypeDef * volatile g_adc_app;   /* APP采集结果。 */
const AD9280_DataTypeDef * volatile g_adc_device;   /* FPGA设备信息。 */
const uint8_t * volatile g_adc_samples;             /* 原始样本缓存。 */
```

在 `USER CODE BEGIN 2` 中临时加入：

```c
g_adc_init_status = AD9280_AppInit();
if (g_adc_init_status == AD9280_OK)
{
  g_adc_capture_status = AD9280_AppCapture(1024U, 1U, 100U);
}
g_adc_app = AD9280_AppGetData();
g_adc_device = AD9280_GetData();
g_adc_samples = AD9280_AppGetSamples();
```

在最后一行设置断点，重新编译和下载。

### 4.3 预期结果

| 观察量 | 正常值 |
|---|---|
| `g_adc_init_status` | `AD9280_OK = 0` |
| `g_adc_capture_status` | `AD9280_OK = 0` |
| `g_adc_device->device_id` | `0xAD928001` |
| `g_adc_device->firmware_version` | `0x00010001` |
| `g_adc_device->sample_clock_hz` | `32000000` |
| `g_adc_app->initialized` | `1` |
| `g_adc_app->sample_count` | `1024` |
| `g_adc_app->capture.done` | `1` |
| `g_adc_app->capture.captured_count` | `1024` |
| `g_adc_app->capture.stored_sample_rate_hz` | 约 `32000000` |
| `g_adc_app->capture.overrange` | `0` |

100 kHz 输入在 32 MSPS 下每周期约 320 点，1024 点缓存应显示约 3.2 个正弦周期。模块按理想 `±5 V` 量程估算时，`1 Vpp、0 V 偏置` 的原始码大约在 `115～141`，平均值约 `128`；实际电位器与模拟前端会带来偏差，应以两点实测校准为准。

### 4.4 抽取与触发

抽取测试可把信号源改为 `1 kHz`，调用：

```c
g_adc_capture_status = AD9280_AppCapture(1024U, 1000U, 100U);
```

此时缓存采样率约 32 kSPS，1024 点应含约 32 个周期。

上升沿阈值触发测试：

```c
g_adc_capture_status = AD9280_AppCaptureTriggered(
    1024U, 1U, AD9280_TRIGGER_RISING, 0.0f, 100U);
```

预期 `triggered=1、waiting_trigger=0、done=1`。本实现没有预触发，所以不会保留阈值越过之前的波形。

### 4.5 故障定位

| 现象 | 优先检查 |
|---|---|
| 初始化返回 `AD9280_ERROR_LINK` | GND、CS、SCK、MOSI、MISO是否一一对应；是否同时接了两块MCU |
| 返回 `AD9280_ERROR_DEVICE` | QSPI 中是不是本集成工程的 `BOOT_PS7.bin` |
| 返回 `AD9280_ERROR_NOT_READY` | FPGA是否完成配置、模块是否正确插入、ACLOCK链路是否存在 |
| 触发采集超时 | 输入是否真的跨过阈值、触发边沿与偏置是否正确 |
| 数据固定在约128 | S2没有信号、线缆断路或输入本身为0 V |
| 大量0/255或`overrange=1` | 输入幅度/偏置过大、模块前端饱和或共地错误 |
| 逻辑分析仪无正确响应 | 每帧CS应持续拉低6字节；MISO前两字节应为 `A5 5A` |

## 5. 高速 DAC 与 ADC 闭环测试

`AD9708_AppInit()` 成功后会把 DAC 停在中点码并关闭交流输出，因此 S1 没有波形是正常初始化现象。

先执行原始码测试：

```c
(void)AD9708_AppInit();
(void)AD9708_AppOutputCalibrationCode(0U);
(void)AD9708_AppOutputCalibrationCode(128U);
(void)AD9708_AppOutputCalibrationCode(255U);
```

三次应得到单调上升的直流电压。分别测量码 0 和码 255 的真实电压，但不要用标称值覆盖已有实测校准。需要电压接口时，在本次上电调用：

```c
(void)AD9708_AppSetVoltageCalibration(实测码0电压, 实测码255电压);
(void)AD9708_AppOutputSine(100000.0f, 合法峰峰值, 合法中心电压);
```

示波器 S1 应看到约 100 kHz 正弦。再用 BNC 短线把 S1 接到 S2，执行 1024 点、不抽取的 ADC 采集，应得到约 3.2 个周期；MCU 读取 BRAM 时 S1 波形应持续，不应中断。

## 6. TI 工程基础测试

TI 工程不是 STM32H743 的等容量替代品，而是按 MSPM0G3507 的 32 MHz Cortex-M0+、128 KB Flash 和 32 KB SRAM 实现的轻量控制端。当前可用功能如下：

当前 `Src/main.c` 已切换为分项实板测试入口，默认运行板载 ADC/FFT；完整的项目选择、烧录和逐项接线优先按 `TI_MSPM0G3507/TI板卡全功能测试.md` 执行。

| 类别 | 已实现功能 | 真实边界 |
|---|---|---|
| 板级基础 | PB26 心跳灯、UART0 调试、1 ms 时基 | 裸机非阻塞主循环 |
| 片上 ADC | PB25 单通道、12 bit、512 点、电压换算 | 软件定时单次采集，最高 20 kSPS，无 DMA 连续流 |
| 轻量 DSP | 512 点 FFT/谐波、测量、FIR/IIR、相关、拟合、IQ、AM/FM、PLL/Costas、SoftPLL、自适应统计 | 处理已在 RAM 中的一帧数据；规模和实时性受 M0+/32 KB SRAM 限制 |
| 外接慢速 DAC/ADC | AD5687、DAC8831、ADS8363 | 软件 SPI；DAC 软件波形最高 1000 点/秒 |
| DDS | AD9910 单频、相位、16～256点RAM波形和扫频 APP | 需要外接 DDS 电源、参考时钟和对应控制线 |
| 高速 AD/DA | AD9708 125 MSPS、AD9280 32 MSPS | 高速逻辑在 FPGA；TI 只配置和读 BRAM |
| FPGA ProMax | 8 路 DDC/CIC 功率、4×32 匹配滤波 | 160 个并行乘法在 FPGA；TI 不搬运 32 MSPS 流，当前无 FPGA FFT |

### 6.1 构建

```powershell

powershell -ExecutionPolicy Bypass -File .\build_keil.ps1
```

脚本会等待 Keil 完整结束，并在出现任何错误或警告时返回失败。正常结果为 `0 Error(s), 0 Warning(s)`。工程中保留的可烧录文件是：

重新构建时，Keil 还会在 `Keil/Objects` 生成同名中间产物；该目录可安全清理，不影响保留的 `output` 镜像。

### 6.2 烧录

1. 用外部XDS110-ETP连接MSP-LITO的J3，再将XDS110 USB连接电脑。
2. 打开 TI UniFlash，选择或自动识别 `MSPM0G3507 + XDS110`。
3. 在 Program 页面选择上面的 `.hex`。
4. 执行 Load Image，确认 Verify 成功后复位板卡。

### 6.3 最小系统预期现象

- PB14 对应MSP-LITO板载D1，PB21对应板载S2按键。
- UART0 为 PA10 TX、PA11 RX、115200-8-N-1，通过J3接入XDS110回传串口。
- 默认 `NUEDC_ENABLE_EXTERNAL_MODULE_INIT=0`，不会初始化未接的 AD9910、AD5687、DAC8831 或 ADS8363 协议；四个低有效片选保持高电平，不产生总线事务。
- 默认 `NUEDC_ENABLE_FPGA_MODULE_INIT=0`。当前分项测试程序只初始化所选项目；选择 `TI_TEST_FPGA_DAC_ADC` 后，FPGA与接线正确时串口应出现：

```text
FPGA AD9708=0, AD9280=0
```

若 BX71 未接，返回非零是链路检测结果，不应影响 LED 和主循环继续运行。

### 6.4 TI 片上 ADC 测试

1. 信号源地接 TI 板 GND，输出接 `PB25 / ADC0_CH4 / J2-15`；禁止输入负电压或超过3.3 V。
2. 设置信号源为 `1 kHz、1.0 Vpp、1.65 V 偏置`，先用示波器确认电压范围约为 1.15～2.15 V。
3. 下载默认 TI 固件并复位。初始化阶段会以 20 kSPS 采集 512 点，主循环第一次调用 `ADC_Sample_Process()` 后完成电压换算。
4. 调试器观察 `ADC_Sample_GetRawData()` 和 `ADC_Sample_GetVoltageData()` 返回的两个缓存。

预期 512 点中约有 25.6 个正弦周期；理想 3.3 V 参考下，原始码最小值约 `1427`、最大值约 `2667`、平均值约 `2048`，换算电压约为 1.15～2.15 V。板载 VDDA、信号源误差和ADC增益误差会造成小幅偏差。

能力边界可用以下返回值确认：采样率超过 20 kSPS、最高谐波超过奈奎斯特频率，或请求 `ADC_MODE_CONTINUOUS` 时，`ADC_Sample_Init()` 都应返回 `HAL_ERROR`；这些限制用于避免把未实现的 DMA 连续采样误当成真实功能。

### 6.5 TI 片上 ADC 与轻量 FFT 联测

保持第 6.4 节的 `1 kHz、1.0 Vpp、1.65 V 偏置` 输入，临时包含 `DSP_Header.h`，并在 `NUEDC_PortInit()` 后执行一次：

```c
NUEDC_PortProcess();
ADC_Sample_RunFft();
ADC_Sample_PrintResult();
```

`NUEDC_PortProcess()` 先把已采完的 512 个原始码换算为电压，随后 FFT 分析该帧。调试器观察 `ADC_Sample_GetResult()`：`status` 应为成功、`frame_count` 增加、`fundamental_hz` 应落在 1 kHz 附近。20 kSPS/512 点的原始频率分辨率约 `39.06 Hz`，所以未校准信号源、窗函数和噪声会造成小量偏差。该测试证明 TI 轻量 DSP 能处理片上 ADC 帧，不表示 M0+ 可以实时处理 AD9280 的连续 32 MSPS 流。

### 6.6 TI 完整功能与主要 API

| 功能 | 已实现接口/行为 | 必须遵守的边界 |
|---|---|---|
| 板级基础 | `NUEDC_PortInit/Process`、1 ms时基、PB26心跳、UART0日志 | 裸机主循环；UART0为PA10/PA11 |
| 片上ADC | `ADC_Sample_Init/Start/Process`、原始码/电压缓存、`ADC_Sample_RunFft` | 512点、最高20 kSPS、仅单次模式；默认从P1经板载缓冲输入 |
| CPU轻量DSP | FFT/谐波、测量、FIR/IIR、相关、拟合、IQ、AM/FM、PLL/Costas、SoftPLL、自适应统计 | 处理RAM中的有限帧，不宣称实时处理32 MSPS流 |
| AD5687 | 四路原始码/直流电压；单路正弦、三角、方波、锯齿、任意波和扫频 | 主循环持续调用`AD5687_AppProcess`；软件送点最高1000点/秒 |
| DAC8831 | 原始码、单/双极性直流；正弦、三角、方波、锯齿、任意波和扫频 | 主循环持续调用`DAC8831_AppProcess`；软件送点最高1000点/秒 |
| AD9910 | 单频、幅度、相位；RAM标准/任意波；自动/手动扫频 | TI端RAM为16～256点；实际DDS系统时钟必须与`AD9910_SYSCLK_HZ`一致 |
| ADS8363 | A侧差分通道0/1选择、原始码、电压和帧计数 | 软件20位SPI；当前不读取SDO-B/B侧 |
| 高速AD/DA | 与STM32相同的`AD9708_App*`、`AD9280_App*`接口 | 125/32 MSPS在FPGA内部；TI只配置和读取缓存 |
| FPGA ProMax | 8路DDC功率、4组32抽头模板、匹配峰值/位置和一致快照 | 当前无FPGA FFT/UART/线级CRC；共享SPI0必须串行访问 |
| 串口屏协议库 | `Screen_Init/Process/Refresh`已编译 | 必须由业务层提供`Screen_ContextTypeDef`并调用`Screen_Init`；仅开处理宏不会自动形成业务界面 |

串口屏兼容层与PC调试输出最终都使用UART0 PA10/PA11，二者必须二选一。需要串口屏时先断开或改接XDS110 Backchannel，并由具体题目业务层提供状态上下文；通用集成工程不会自动启动旧题状态机。

## 7. TI 与 FPGA 联动测试

1. 只保留第 3.2 节的 TI～BX71 五根连接。
2. BX71 从 QSPI 启动，TI 板从 XDS110/USB 供电，两板只共地、不互供电。
3. 复位 TI，先确认串口两项状态均为 0。
4. 调试器观察：

```c
AD9708_AppGetData()->device_id         == 0xAD970802
AD9708_AppGetData()->firmware_version  == 0x00020003
AD9708_AppGetData()->dac_ready         == 1
AD9708_AppGetData()->pll_locked        == 1
AD9280_GetData()->device_id            == 0xAD928001
AD9280_GetData()->firmware_version     == 0x00010001
AD9280_GetData()->sample_clock_hz      == 32000000
```

5. 在 `NUEDC_PortInit()` 之后只执行一次：

```c
(void)AD9708_AppOutputCalibrationCode(128U);
(void)AD9280_AppCapture(1024U, 1U, 100U);
```

S1 应为中点附近直流；S2 接 100 kHz、1 Vpp、0 V 偏置信号时，ADC结果与第 4.3 节相同。

TI SPI0 固定为 Mode 0、8 bit、MSB first、1 MHz。逻辑分析仪应看到 PA15 空闲为高，每个寄存器帧期间连续为低，PA12 有约 1 MHz 时钟，MISO每帧前两字节为 `A5 5A`。1024/4096点采样是在 FPGA 内快速完成，TI 逐寄存器回读会明显更慢；约 0.1 s/0.3～0.6 s 属正常量级，不代表 ADC 采样率下降。

## 8. FPGA ProMax 实时分析联测

本节对 STM32H743 和 TI MSPM0G3507 都适用；两块 MCU 的公共 API 相同，底层分别使用 SPI2 和 SPI0。一次只能连接其中一块 MCU。

### 8.1 准备与初始化

1. 先分别完成第 4、5、7 节，确认 ADC、DAC 和 6 字节链路都正常。
2. 用 BNC 短线把 S1 接到 S2；先用示波器确认 S1 幅度不超过约 2 Vpp、中心电压位于 AD9280 模块约 `-5～+5 V` 输入范围。
3. STM32 在业务初始化中先调用 `AD9708_AppInit()` 和 `AD9280_AppInit()`；TI 端选择 `TI_TEST_FPGA_PROMAX` 后，由测试入口初始化 AD9280 和 ProMax。一次只运行一个测试项目，无需在外面重复初始化。
4. AD9708 两点校准保存在 MCU RAM，复位或重新上电后 `valid` 会恢复为 0；工程不会猜测或覆盖你的实测参数。每次冷启动都必须先把你已经校准好的码 0、码 255 实测电压重新传给 `AD9708_AppSetVoltageCalibration()`，或从你自己的非易失存储中读出后装载。随后以两端中点为偏置、较小合法 Vpp 输出 1 MHz 正弦。

下面的辅助函数不包含也不改写任何固定校准数值：调用者把自己已有的两个实测值传入。幅度最多取 1 Vpp；调用前仍应先用示波器确认所得中心电压和幅度在 S2 允许范围内，并检查函数返回值：

```c
/* 装载已有两点校准并启动1 MHz闭环测试信号。 */
static AD9708_StatusTypeDef ProMax_StartLoopbackTone(float code0_voltage_v,
                                                      float code255_voltage_v)
{
  const AD9708_VoltageCalibrationTypeDef *cal;
  AD9708_StatusTypeDef status;
  float span_v;
  float amplitude_vpp;
  float offset_v;

  status = AD9708_AppSetVoltageCalibration(code0_voltage_v,
                                            code255_voltage_v);
  if (status != AD9708_OK)
  {
    return status;
  }

  cal = AD9708_AppGetVoltageCalibration();
  if ((cal == NULL) || (cal->valid == 0U))
  {
    return AD9708_ERROR_CALIBRATION;
  }

  span_v = cal->code255_voltage_v - cal->code0_voltage_v;
  amplitude_vpp = span_v * 0.2f;
  if (amplitude_vpp > 1.0f)
  {
    amplitude_vpp = 1.0f;
  }
  offset_v = (cal->code255_voltage_v + cal->code0_voltage_v) * 0.5f;
  return AD9708_AppOutputSine(1000000.0f, amplitude_vpp, offset_v);
}
```

调用 `ProMax_StartLoopbackTone(你的码0实测电压, 你的码255实测电压)` 应返回 `AD9708_OK`，示波器确认 S1 正常后再接入 S2。若返回 `AD9708_ERROR_CALIBRATION` 或 `AD9708_ERROR_PARAM`，不要继续 ProMax 闭环测试，应先检查传入的实测值；此步骤只重新装载原值，不要求重新标定，也不修改工程中的校准参数。

| 观察量 | 预期值 |

| `s_promax.version` | `0x00010000` |
| `s_promax.capability` | `0x09200408` |

### 8.2 可直接使用的 DDC/匹配测试代码

在业务文件中加入 `fpga_promax_link.h`，定义以下测试状态。32 点表是 `1 MHz @ 32 MSPS` 的 Q1.16 正弦模板；`reverse=1` 使其成为匹配滤波系数。

```c
static FpgaPromax s_promax;                    /* FPGA实时DSP句柄。 */
static FpgaPromax_AllResults s_promax_results; /* FPGA压缩结果。 */
static uint32_t s_promax_status;                /* FPGA实时DSP原始状态。 */
volatile FpgaPromax_Result g_promax_test_result; /* ProMax闭环测试返回值。 */

static const int32_t s_sine_1m_q16[32] = {     /* 1 MHz正弦匹配模板。 */
       0,  12785,  25079,  36410,  46341,  54491,  60547,  64277,
   65536,  64277,  60547,  54491,  46341,  36410,  25079,  12785,
       0, -12785, -25079, -36410, -46341, -54491, -60547, -64277,
  -65536, -64277, -60547, -54491, -46341, -36410, -25079, -12785
};

/* 配置八路DDC和一组模板，随后读取一次一致快照。 */
static FpgaPromax_Result ProMax_RunLoopbackTest(void)
{
  FpgaPromax_Result result;
  uint8_t channel;

  result = FPGA_ProMax_Init(&s_promax);
  if (result != FPGA_PROMAX_OK)
  {
    return result;
  }
  result = FpgaPromax_SetRun(&s_promax, 0U);
  for (channel = 0U;
       (channel < FPGA_PROMAX_DDC_CHANNELS) &&
       (result == FPGA_PROMAX_OK);
       channel++)
  {
    result = FpgaPromax_SetDdcFrequency(&s_promax,
                                        channel,
                                        (uint32_t)(channel + 1U) * 1000000U,
                                        32000000U);
  }
  if (result == FPGA_PROMAX_OK)
  {
    result = FpgaPromax_LoadMatchedTemplate(&s_promax,
                                            0U,
                                            s_sine_1m_q16,
                                            32U,
                                            1U);
  }
  if (result == FPGA_PROMAX_OK)
  {
    result = FpgaPromax_ClearState(&s_promax);
  }
  if (result == FPGA_PROMAX_OK)
  {
    result = FpgaPromax_SetRun(&s_promax, 1U);
  }
  if (result == FPGA_PROMAX_OK)
  {
    result = FpgaPromax_ReadAllResults(&s_promax,
                                       &s_promax_results,
                                       100U);
  }
  if (result == FPGA_PROMAX_OK)
  {
    result = FpgaPromax_GetStatus(&s_promax, &s_promax_status);
  }
  return result;
}
```

确认 `ProMax_StartLoopbackTone(...)` 返回 `AD9708_OK` 且示波器上的 S1 波形正常后，在业务初始化中实际执行一次；仅复制函数定义不会启动硬件：

```c
g_promax_test_result = ProMax_RunLoopbackTest();
```

### 8.3 预期结果与判定

- `ProMax_RunLoopbackTest()` 应返回 `FPGA_PROMAX_OK`；`s_promax_status & 0x7U` 应为 `0x7U`，表示运行、功率和匹配结果均有效。
- 1 MHz 输入对应 DDC0。`band_power[0]` 应明显大于配置为 2～8 MHz 的其余通道；绝对数值是定点功率，不是直接的 V²。建议实板先以“目标通道大于最大非目标通道 4 倍”作为基础判据，再按模拟链路噪声实测收紧。
- `matched_score[0]` 的绝对值应明显非零；模板 1～3 未装载时保持零模板，其分数应接近 0。分数正负取决于输入起始相位。
- `matched_peak_index[0]` 是本次清理后出现最大绝对相关值的位置。相同峰值不会不断刷新；再次 `FpgaPromax_ClearState()` 后它从新一轮样本重新统计。
- `power_generation` 和 `score_generation` 是结果代次；`FpgaPromax_ReadAllResults()` 会冻结同一快照读取并在结束后自动解冻。
- 正常状态的 FIFO overflow 位（状态 bit3）应为 0。若为 1，先检查 ADC 时钟、FPGA 镜像和复位，再清状态复测。

逻辑分析仪在线上仍应看到原来的 6 字节帧与 `A5 5A` ACK，而不是 9 字节 ProMax 帧。9 字节帧、CRC 和序号由 MCU 适配层在本地生成/校验；当前能力位不宣称线级 CRC。

### 8.4 常见故障

| 现象 | 优先检查 |
|---|---|
| 初始化为 `FPGA_PROMAX_E_WRONG_DEVICE` | QSPI 是否烧入本次新 `BOOT_PS7.bin`；读取 ID 是否为 `0x4258504D` |
| 初始化为传输错误 | MCU 选择是否唯一、五根线、共地、CS整帧低电平、SPI Mode 0 |
| 一直等待结果超时 | 是否已 `SetRun(1)`；S2/AD9280 时钟是否工作；状态 bit6 的 ADC ready 是否为1 |
| 八路功率都接近0 | S1→S2是否连通、ADC码是否围绕128变化、DDC采样率是否固定传入32 MHz |
| DDC0不占优 | 实际信号是否为1 MHz、S1是否削顶、采样时钟或NCO频率参数是否错误 |
| 匹配分数为0 | 模板是否成功写入、是否在装载后清状态并启动、ADC输入是否以中点码为零 |

## 9. TI 外接模块逐项测试

在 Keil 的 `Options for Target -> C/C++ -> Preprocessor Symbols` 中一次只添加需要测试的开关，例如 `NUEDC_ENABLE_AD9910=1`。静态电压输出可以共用数据线；不要同时启动 AD5687 与 DAC8831 的软件波形任务。下表统一使用MSP-LITO的J1/J2排针编号。

### 9.1 AD9910

| AD9910信号 | TI引脚 | MSP-LITO位置 |
|---|---|---|
| SCK | PB20 | J2-23 |
| SDIO | PB2 | J1-11 |
| PWR | PB13 | J1-26 |
| DROVER | PB1 | J1-9 |
| DRCTL | PA8 | J1-15 |
| DRHOLD | PA16 | J2-31 |
| RESET | PB4 | J1-13 |
| PF0 | PB5 | J1-14 |
| PF1 | PA17 | J2-30 |
| PF2 | PA9 | J1-16 |
| OSK | PB7 | J1-20 |
| IO_UPDATE | PB8 | J1-21 |
| CS | PB9 | J1-22 |

PA19/PA20 保留给 SWD，不再被 AD9910 占用。PB1、PB2 和 PB4 无默认板载负载；原有 PA0、PB21、PA6 映射已分别避开板载 LED/上拉、用户按键 S2 和 40 MHz 晶振网络。完成模块参考时钟和电源连接后调用：

```c
(void)AD9910_AppOutputSine(1000000U, 8192U);
```

预期 DDS 输出约 1 MHz 正弦。TI 端 AD9910 RAM 波形点数范围为 16～`AD9910_RAM_POINTS`，当前上限 256 点；传入 1024 点会正确返回参数错误。若频率按固定比例偏差，先核对 AD9910 实际系统时钟是否与代码中的 `AD9910_SYSCLK_HZ` 一致。
